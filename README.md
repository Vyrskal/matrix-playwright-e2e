# E2E-тест для веб-клиента Matrix (Element)

Данный репозиторий содержит автоматизированный E2E-тест для мессенджера на базе протокола Matrix (на примере Element), написанный с использованием Playwright и TypeScript.

## Реализованные шаги:
1. Авторизация в веб-клиенте.
2. Создание новой комнаты.
3. Отправка текстового сообщения с уникальным timestamp.
4. Проверка отображения отправленного сообщения в ленте.
5. Дополнительная проверка состояния сессии (на основе `localStorage`).

## Архитектура и структура
Тесты спроектированы на основе паттерна **Page Object Model (POM)** для разделения логики поиска элементов и самих сценариев:
- `page-objects/` — Классы взаимодействия со страницами.
  - `LoginPage.ts` — логика авторизации.
  - `ChatPage.ts` — логика работы со списком комнат и отправкой сообщений.
- `tests/` — Спецификации тестов.
  - `chat.spec.ts` — основной тестовый сценарий (включает шаги `test.step` для детализированных отчетов).

## Инструкция по запуску

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Установите браузеры Playwright:
   ```bash
   npx playwright install chromium
   ```

3. Настройте конфигурацию:
   Скопируйте `.env.example` в `.env` и укажите данные вашего тестового аккаунта:
   ```bash
   cp .env.example .env
   ```
   *Важно:* Для теста потребуется существующий аккаунт на сервере Matrix. По умолчанию используется `https://app.element.io`, но это можно переопределить через `BASE_URL`.

4. Запуск теста:
   - В headless режиме (тихий запуск):
     ```bash
     npm run test
     ```
   - С открытием UI браузера (headed mode):
     ```bash
     npm run test:headed
     ```
   - В режиме Playwright UI Mode:
     ```bash
     npm run test:ui
     ```

**Фактически затраченное время:** Около 1 часа на развертывание проекта, проектирование POM, написание тестового сценария и подготовку документации (README).
